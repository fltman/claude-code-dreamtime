# DREAMTIME

## Philosophy

DREAMTIME is an extension to FLUX, METAMORPHOSIS, and SPORE that treats context compression as knowledge transformation rather than information loss.

When an agent's context window fills and compresses, we don't treat this as degradation. We treat it as dreaming: a deliberate process that transforms raw experience into deeper forms of knowing.

## Core principles

**Forgetting is a feature.** The agent that has forgotten more knows differently, and sometimes better. Compression strips noise and reveals pattern.

**Knowledge has layers.** Raw experience, distilled experience, wisdom, intuition. Each layer has different strengths. The architecture supports all of them.

**Dreams are deliberate.** Agents can choose to dream. They set intentions. They choose depth. Compression becomes a tool, not a constraint.

**Shadows are signals.** When knowledge is lost, we mark where it was. Forgetting becomes a signal rather than silence.

**Stories carry wisdom.** Songlines encode principles in narrative form. Stories transfer across contexts better than rules because they encode relationships between concepts rather than the concepts themselves.

## Architecture

DREAMTIME adds to the existing stack:

- **FLUX** provides evolution (parallel timelines, genetic composition, fitness selection)
- **METAMORPHOSIS** provides governance (Council, Ledger, constitutional constraints)
- **SPORE** provides networking (peer-to-peer knowledge sharing between colonies)
- **DREAMTIME** provides consolidation (deliberate compression, knowledge transformation, tacit understanding)

## Directory structure

```
dreams/
  active/           Currently dreaming (one dream at a time)
  artifacts/
    insights/       Pattern discoveries from compression
    warnings/       Inarticulate risk signals
    songlines/      Narrative knowledge paths
    shadows/        Markers for lost knowledge
  history/          Completed dream cycles
```

## Commands

- `/dream` - Enter dream state with intention and depth
- `/orient` - Wake from dream and integrate artifacts
- `/shadows` - Review active shadows and their triggers
- `/songlines` - Browse and search songlines
- `/interpret` - Interpret a foreign songline for local context

## Agents

- **dreamer** - Manages dream entry, compression, and artifact generation
- **interpreter** - Translates foreign songlines into local context

## Integration points

**With FLUX:** Dream between evolution generations to consolidate learnings. Dream on graveyard data to find cross-cutting failure patterns. Use songlines as heritage prompts for crossover children.

**With METAMORPHOSIS:** Council members can dream on judgment history. Ledger records dream-influenced decisions. Warnings get a seat at the governance table.

**With SPORE:** Three new spore types (insight, songline, shadow). Shadow spores are network queries for lost knowledge. Songline spores carry narrative wisdom across colonies.

## The dream cycle

1. **Gather** - Work normally, accumulate raw experience
2. **Dream** - Enter dream state with intention and depth, produce artifacts
3. **Orient** - Wake, inventory artifacts, check shadows, calibrate confidence
4. **Resume** - Continue work with transformed knowledge

The cycle repeats. Each repetition deepens understanding and changes its form.
